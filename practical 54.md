
# Practical Guide for Understanding Call Detail Records (CDR)

Call Detail Records (CDR) are records maintained by telecom service providers that contain details of communication activities such as calls, SMS, and internet usage. CDR analysis is widely used in cybercrime investigations, digital forensics, and law enforcement for tracking communication patterns and device usage.

---
# Introduction to CDR

A Call Detail Record (CDR) is a metadata record generated by telecom operators whenever a user performs activities such as:

* Making or receiving calls
* Sending or receiving SMS
* Using mobile internet
* Roaming activities
* International communication

CDR does not store the actual conversation or message content. Instead, it stores information about the communication event.

# Types of CDR

## 1. Normal CDR

Contains communication records of:

* Accused persons
* Suspects
* Victims

### Information Included

* Calling number
* Called number
* Date and time
* Duration
* Tower location
* IMEI
* IMSI
* Call type

---

## 2. Site CDR / BTS CDR

Contains records of all devices connected to a specific tower or BTS (Base Transceiver Station).

### Uses

* Crowd analysis
* Location tracing
* Suspect identification near crime scene

---

## 3. IMEI CDR

Tracks all SIM cards used in a particular mobile handset.

### IMEI

IMEI stands for:

**International Mobile Equipment Identity**

* Usually 14 or 15 digits
* Unique identifier of a mobile handset

### Uses

* Tracking stolen devices
* Identifying SIM swapping
* Device usage analysis

* Internet sessions

# Important CDR Concepts

## BTS (Base Transceiver Station)

A BTS is a telecom tower that handles communication between mobile devices and the network.

### Tower Information

* Each tower has a Cell ID
* Used for location tracking

---

# Cell ID Structure

Cell ID is generally represented as:

```text
MCC-MNC-LAC-CID
```

Example:

```text
404-20-11-34273
```

### Explanation

| Field | Meaning                  |
| ----- | ------------------------ |
| 404   | Country code for India   |
| 20    | Operator code (Vodafone) |
| 11    | LAC (Location Area Code) |
| 34273 | CID (Cell ID)            |

---

# Mobile Number Structure

Example Mobile Number:

```text
9958452532
```

### Breakdown

| Part  | Meaning           |
| ----- | ----------------- |
| 99584 | MSC Code          |
| 52532 | Subscriber Number |

# LRN

LRN stands for:

**Location Routing Number**

* Usually a 4-digit code
* Helps route calls between telecom networks

* Calls with duration 0 or 1 second generally indicate SMS activity or missed calls

# Location Tracking in CDR

## When Mobile is Switched Off

* Live location is not available
* Last connected tower may be available

## When Mobile is Switched On

### Available Information

* Last tower location
* Current live tower location
* LBS (Location Based Service)

### LBS Information

Provides:

* Latitude
* Longitude
* Approximate live location

---

# Important Notes About Normal CDR

In Normal CDR:

* Intermediate tower locations may not always be available
* Usually first and last tower locations are recorded

---

# CDR Availability

* Telecom companies generally provide Normal CDR up to 1 year
* Older records may be archived in tape drives or backup systems

# Example Investigation Scenario

### Scenario

A cybercrime investigation team receives a complaint regarding threatening phone calls.

### Investigation Steps

1. Obtain Normal CDR of the suspect number
2. Identify:

   * Calling patterns
   * Frequent contacts
   * Tower locations
3. Check IMEI CDR to determine whether multiple SIM cards were used in the same handset
4. Analyze BTS CDR near the crime location
5. Use IPDR to identify internet activity linked to the suspect
6. Examine ILD CDR for international communication

### Result

The investigation helps establish:

* Suspect movement
* Device ownership
* Communication network
* Internet activity
* Possible accomplices
