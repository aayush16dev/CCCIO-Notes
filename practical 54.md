# Practical Guide for Understanding Call Detail Records (CDR)

Call Detail Records (CDR) are records maintained by telecom service providers that contain details of communication activities such as calls, SMS, and internet usage. CDR analysis is widely used in cybercrime investigations, digital forensics, and law enforcement for tracking communication patterns and device usage.

---

# Introduction to CDR

A **Call Detail Record (CDR)** is a metadata record automatically generated by a telecom operator's systems every time a subscriber performs a communication activity. The telecom network's switching infrastructure — specifically the MSC (Mobile Switching Center) — creates these records in real time and stores them for billing, network management, and lawful interception purposes.

CDR is generated for activities such as:

- Making or receiving voice calls
- Sending or receiving SMS messages
- Using mobile internet (GPRS/3G/4G/5G data sessions)
- Roaming on other networks
- International communication

> **Important:** CDR does **not** store the actual content of calls or messages. It only stores metadata — information *about* the communication event (who, when, where, how long), not what was said or written.

# Telecom Network Architecture

Understanding how CDR is generated requires understanding the underlying network architecture. When a call is made, it travels through several components:

```
📱 Mobile Device (Handset)
        ⬇
📡 BTS — Base Transceiver Station (Mobile Tower)
        ⬇
BSC — Base Station Controller
        (Controls multiple BTS units, manages radio resources and traffic)
        ⬇
MSC — Mobile Switching Center
        (Handles call routing, roaming, and CDR generation)
        ⬇
HLR / VLR — Subscriber Databases
        ⬇
Other Network / Recipient
```

### Component Breakdown

**BTS (Base Transceiver Station)**
The physical mobile tower. It receives and transmits radio signals between the mobile device and the network. Key facts:
- Each BTS typically has **3 sectors**, each covering **120°** of a full 360° circle
- Some BTS installations have **4 sectors** covering **90°** each
- Each sector has its own unique Cell ID
- Connects directly to the BSC

**BSC (Base Station Controller)**
Controls multiple BTS units in a geographic area. Manages radio resource allocation and handover between towers when a device moves.

**MSC (Mobile Switching Center)**
The core of CDR generation. The MSC routes calls, manages roaming, handles billing triggers, and writes the CDR entry when a communication event completes.

**HLR (Home Location Register)**
A permanent database containing subscriber information:
- Mobile number (MSISDN — Mobile Station International Subscriber Directory Number)
- Subscriber profile and service permissions
- Network details

**VLR (Visitor Location Register)**
A temporary database that stores details of subscribers currently roaming or visiting a new location area. Gets updated every time the device moves into a new zone.

# Call Types

Calls and communications are classified into two primary categories:

| Type | Code | Meaning |
|---|---|---|
| Mobile Originated | **MO** | Communication initiated by the user (outgoing call, sent SMS) |
| Mobile Terminated | **MT** | Communication received by the user (incoming call, received SMS) |

### Full list of call/event types recorded in CDR:

- Incoming voice call
- Outgoing voice call
- Incoming SMS
- Outgoing SMS
- Roaming call
- International call
- Data session (GPRS/3G/4G)

# Types of CDR

## 1. Normal CDR

The most commonly requested CDR type in investigations. It contains the complete communication history of a specific subscriber (accused, suspect, or victim).

### Fields Included

| Field | Description |
|---|---|
| A_number | Calling party (the one who initiated the call) |
| B_number | Called party (the recipient) |
| Date | Date of the communication event |
| Time | Time of the communication event |
| Duration | Length of the call in **seconds** |
| Cell_ID | Tower identifier where the call was connected |
| IMEI | Device identifier of the handset used |
| IMSI | SIM card identifier |
| Call Type | MO / MT / SMS / Data |

> **Note on Duration:** A duration of **0 or 1 second** typically indicates a missed call, a rejected call, or an SMS event rather than a completed voice call.

## 2. Site CDR / BTS CDR

Also called a **Tower Dump**. Instead of focusing on one subscriber, this CDR contains records of *all* devices that connected to a specific tower or BTS during a given time window.

### Uses in Investigation

- **Crowd analysis** — identifying all people present at a location during an event
- **Crime scene analysis** — finding devices that were near the crime scene at the time of the incident
- **Suspect identification** — narrowing down suspects from hundreds of devices to a shortlist

## 3. IMEI CDR

Tracks all SIM cards that have ever been used in a specific mobile handset, identified by its IMEI number.

### What is IMEI?

**IMEI** stands for **International Mobile Equipment Identity**.

- A unique 14 or 15-digit number assigned to every mobile handset
- Hardcoded into the device hardware
- Cannot be changed without tampering (though some devices are illegally re-flashed)

### IMEI CDR Uses

- **Tracking stolen devices** — identifying which SIMs were inserted after theft
- **Detecting SIM swapping** — finding multiple SIMs used in one device
- **Linking suspects** — connecting different phone numbers to the same physical handset

![](<Screenshot 2026-05-22 123256.png>)

---

## 4. Normal GPRS CDR

Contains internet usage records for a specific subscriber.

**GPRS** stands for **General Packet Radio Service** — the technology that enables mobile data connectivity.

### Fields Included

- Internet session start and end times
- Total data usage (upload/download in bytes)
- IP address allocated during the session
- Cell ID of the tower used for the data session

![](<Screenshot 2026-05-22 123753.png>)

---

## 5. Tower GPRS CDR

Contains internet usage records associated with a specific tower, similar to how BTS CDR works for voice calls.

### Uses

- Analyzing internet activity originating from a specific geographic location
- Identifying devices that accessed data services near a crime scene
- Tower-based data traffic analysis

![](<Screenshot 2026-05-22 123942.png>)

---

## 6. IPDR (Internet Protocol Detail Record)

A more granular internet record that maps a subscriber's IP address to specific application-level activities.

### What IPDR Reveals

- Gmail login sessions
- Facebook or social media access
- Messaging app usage (WhatsApp, Telegram)
- Specific websites visited
- IP address mapping to user identity at a given timestamp

IPDR is critical for cybercrime investigations where the suspect operated online and needs to be traced from an IP address back to a physical subscriber identity.

## 7. ILD CDR

**ILD** stands for **International Long Distance CDR**.

Contains records of all international calls made or received by a subscriber. Used when investigating cross-border criminal activity, international fraud, or terrorism-related communication.

# Key Identifiers in CDR

## Cell ID Structure

Every tower sector is globally identified using the **CGI (Cell Global Identity)** format:

```
MCC - MNC - LAC - CID
```

**Example:**
```
404 - 20 - 11 - 34273
```

| Field | Value | Meaning |
|---|---|---|
| MCC | 404 | Mobile Country Code — India |
| MNC | 20 | Mobile Network Code — Vodafone |
| LAC | 11 | Location Area Code |
| CID | 34273 | Cell ID (unique tower sector identifier) |

> **LAC (Location Area Code):** Groups multiple BTS towers into a location area. LAC + Cell ID together uniquely identify a specific tower sector.

> **CGI (Cell Global Identity):** The full combination of MCC + MNC + LAC + Cell ID gives the globally unique identity of a tower sector.

**India-specific values:**
- MCC = **404** for all Indian operators
- MNC = **45** for Vodafone Idea (VI)

---

## IMSI Structure

**IMSI** stands for **International Mobile Subscriber Identity**. It is stored on the SIM card and uniquely identifies the subscriber on the mobile network.

| Component | Full Form | Purpose |
|---|---|---|
| MCC | Mobile Country Code | Identifies the country where the phone is being used |
| MNC | Mobile Network Code | Identifies the telecom operator |
| MSIN | Mobile Subscriber Identification Number | Uniquely identifies the individual user |

---

## Mobile Number Structure

Indian mobile numbers follow a structured format. For example:

```
9958452532
```

| Part | Value | Meaning |
|---|---|---|
| MSC Code | 99584 | Identifies the MSC serving that number |
| Subscriber Number | 52532 | Unique subscriber identifier under that MSC |

---

## LRN (Location Routing Number)

**LRN** stands for **Location Routing Number**.

- Usually a 4-digit code
- Used to route calls efficiently between different telecom networks (especially during number portability)
- Helps the network determine which operator currently holds the destination number

# Location Tracking Using CDR

CDR provides valuable location information based on which tower the device was connected to at the time of a communication event.

## When the Mobile is Switched OFF

- Real-time live location is **not available**
- The **last recorded tower** before the device was switched off may be available from the final CDR entry

## When the Mobile is Switched ON

Three types of location data may be available:

| Type | Description |
|---|---|
| Last tower location | The most recent tower the device connected to |
| Current live tower location | The tower currently serving the device |
| LBS (Location Based Service) | Provides latitude and longitude for more precise location |

### LBS (Location Based Service)

LBS uses signal triangulation from multiple towers to provide:
- **Latitude** and **Longitude** coordinates
- Approximate live location of the device
- More precise than single-tower Cell ID alone

## Limitations of Normal CDR for Location

- Normal CDR typically records only the **first** and **last** tower used during a call
- Intermediate towers the device passed through during a long call may **not** be recorded
- For precise movement tracking, BTS CDR (tower dump) combined with IMEI CDR provides better results

---

# Handover Process

When a user is in motion (e.g., driving), their device continuously shifts between towers. The **handover process** is the mechanism by which the network seamlessly transfers the active connection from one BTS to another without dropping the call.

- Managed by the BSC
- Prevents call drops during movement
- Each handover event can create an entry in the CDR with an updated Cell ID, enabling movement tracking

# CDR Availability

| CDR Type | Typical Retention Period |
|---|---|
| Normal CDR | Up to **1 year** |
| GPRS / Data CDR | Varies by operator |
| Archived records | May be available on tape drives or backup systems beyond 1 year |

Investigators must submit formal requests (through legal channels) to the telecom operator to obtain CDR within the retention window.

# Example Investigation Scenario

### Scenario

A cybercrime investigation team receives a complaint about threatening phone calls received by a victim.

### Step-by-Step Investigation

1. **Obtain Normal CDR** of the suspect number to analyze calling patterns, contact frequency, and tower locations
2. **Check IMEI CDR** to determine whether multiple SIM cards were used in the same handset — identifying SIM swapping behavior
3. **Analyze BTS CDR** (tower dump) near the crime location and timestamp to find all devices present
4. **Use GPRS CDR** to identify internet sessions linked to the suspect's number
5. **Examine IPDR** to trace specific online activity (social media, messaging apps) to the suspect
6. **Check ILD CDR** if there is suspicion of international communication with accomplices

### What the Investigation Establishes

| Finding | CDR Source |
|---|---|
| Suspect's physical movement | Normal CDR (Cell ID) + BTS CDR |
| Device ownership and SIM swapping | IMEI CDR |
| Communication network and accomplices | Normal CDR (B_number analysis) |
| Online activity and platform usage | IPDR |
| International links | ILD CDR |

# Python Automation of CDR

Manual CDR analysis becomes impractical when dealing with thousands of records. Python — particularly the `pandas` library — allows investigators and analysts to automate CDR processing and extract actionable intelligence quickly and accurately.

|---|---|

## Python CDR Analysis Script

The script below performs 10 standard CDR analysis operations on the dataset. Save `Book1.csv` and `cdr_analysis.py` in the same folder and run it using Python 3.

```python
import pandas as pd

# Load CDR Data
df = pd.read_csv("Book1.csv")

# Show full data
print("Full Data:")
print(df)

# 1. Count total calls
total_calls = len(df)
print("\nTotal Calls:", total_calls)

# 2. Most frequent contact (B_number)
contact = df["B_number"].value_counts()
print("\nMost Contacted Number:")
print(contact)

# 3. Total call duration per B_number
duration = df.groupby("B_number")["Duration"].sum()
print("\nTotal Duration per B_number (seconds):")
print(duration)

# 4. Suspicious calls after 10 PM (22:00)
df["Time"] = pd.to_datetime(df["Time"], format="%H:%M")
late_night = df[df["Time"].dt.hour >= 22]
print("\nLate Night Calls (after 10 PM):")
if not late_night.empty:
    print(late_night)
else:
    print("No late night calls found.")

# 5. Tower/location usage frequency
tower_usage = df["Cell_ID"].value_counts()
print("\nTower Usage:")
print(tower_usage)

# 6. Filter records for a specific suspect number
suspect = 5648524893
suspect_data = df[
    (df["A_number"] == suspect) |
    (df["B_number"] == suspect)
]
print(f"\nAll Records for Suspect ({suspect}):")
print(suspect_data)

# 7. Unique IMEI numbers in the dataset
imei_usage = df["IMEI"].unique()
print("\nUnique IMEI Numbers Found:")
print(imei_usage)

# 8. Total outgoing call duration per A_number (caller activity)
caller_duration = df.groupby("A_number")["Duration"].sum()
print("\nTotal Outgoing Duration per Caller:")
print(caller_duration)

# 9. Number of calls per date
date_calls = df["Date"].value_counts()
print("\nCalls per Date:")
print(date_calls)

# 10. Tower movement sequence (for tracking movement)
tower_sequence = df[["A_number", "Cell_ID", "Date", "Time"]]
print("\nTower Movement Log:")
print(tower_sequence)

print("\nCDR Analysis Completed.")
```

## What Each Analysis Step Does

| Step | Operation | Investigative Purpose |
|---|---|---|
| 1 | Count total calls | Understand the volume of communication |
| 2 | Most frequent B_number | Identify the most contacted person — potential associate |
| 3 | Total duration per B_number | Find who the suspect spent the most time talking to |
| 4 | Late night calls (after 10 PM) | Flag suspicious communication outside normal hours |
| 5 | Tower usage frequency | Identify locations the suspect frequented most |
| 6 | Filter by suspect number | Isolate all calls involving a specific person of interest |
| 7 | Unique IMEIs | Detect multiple devices or SIM swapping activity |
| 8 | Outgoing duration per caller | Identify who was making the most outgoing calls |
| 9 | Calls per date | Spot unusual spikes in communication activity on specific dates |
| 10 | Tower movement sequence | Reconstruct the movement of a suspect over time |
